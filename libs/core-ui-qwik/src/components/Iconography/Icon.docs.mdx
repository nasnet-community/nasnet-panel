# Icon Component

The Icon component provides a consistent way to display icons across the Connect application. It leverages the @qwikest/icons library with HeroIcons to maintain a cohesive visual language.

## Usage

```tsx
import { Icon, HomeIcon } from "@/components/Core/Iconography";

export default component$(() => {
  return <Icon icon={HomeIcon} />;
});
```

## Props

### Required Props

| Prop   | Type                 | Description                                                                   |
| ------ | -------------------- | ----------------------------------------------------------------------------- |
| `icon` | `QRL<() => JSXNode>` | The icon to display - should be a QRL created with $() from an icon component |

### Optional Props

| Prop         | Type                                                                                                           | Default     | Description                                                                                |
| ------------ | -------------------------------------------------------------------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------ |
| `size`       | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl' \| '2xl'`                                                                | `'md'`      | Size of the icon                                                                           |
| `color`      | `'inherit' \| 'current' \| 'primary' \| 'secondary' \| 'success' \| 'warning' \| 'error' \| 'info' \| 'muted'` | `'current'` | Color variant for the icon                                                                 |
| `fixedWidth` | `boolean`                                                                                                      | `false`     | Whether the icon should have a fixed width (useful for alignment)                          |
| `class`      | `string`                                                                                                       | `''`        | Additional CSS classes to apply to the icon                                                |
| `label`      | `string`                                                                                                       | `undefined` | Accessible label for the icon (for screen readers). Leave empty if icon is decorative only |

## Icon Sizes

The Icon component supports various sizes to fit different contexts:

```tsx
<Icon icon={HomeIcon} size="xs" /> // Extra small - 12px
<Icon icon={HomeIcon} size="sm" /> // Small - 16px
<Icon icon={HomeIcon} size="md" /> // Medium - 20px (default)
<Icon icon={HomeIcon} size="lg" /> // Large - 24px
<Icon icon={HomeIcon} size="xl" /> // Extra large - 32px
<Icon icon={HomeIcon} size="2xl" /> // 2X Large - 40px
```

## Icon Colors

Icons can be styled with different semantic colors:

```tsx
<Icon icon={HomeIcon} color="primary" />
<Icon icon={HomeIcon} color="secondary" />
<Icon icon={HomeIcon} color="success" />
<Icon icon={HomeIcon} color="warning" />
<Icon icon={HomeIcon} color="error" />
<Icon icon={HomeIcon} color="info" />
<Icon icon={HomeIcon} color="muted" />
<Icon icon={HomeIcon} color="current" /> // Inherits color from text (default)
```

## Icon Variants

The Connect icon system includes both outline and solid variants for most icons:

```tsx
// Outline variant (default)
<Icon icon={HomeIcon} />

// Solid variant
<Icon icon={HomeSolidIcon} />
```

## Fixed Width

When aligning icons in a list or navigation, use the `fixedWidth` prop to ensure consistent spacing:

```tsx
<div class="flex flex-col">
  <div class="flex items-center">
    <Icon icon={HomeIcon} fixedWidth />
    <span>Home</span>
  </div>
  <div class="flex items-center">
    <Icon icon={SettingsIcon} fixedWidth />
    <span>Settings</span>
  </div>
</div>
```

## Accessibility

For accessibility, provide a descriptive `label` prop for icons that convey meaning. Leave the label empty for decorative icons:

```tsx
// Semantic icon with label for assistive technologies
<Icon icon={ErrorIcon} color="error" label="Error occurred" />

// Decorative icon without label
<Icon icon={DecorativeIcon} />
```

## Adding New Icons

If you need additional icons that aren't in the core set:

1. Import the icon from '@qwikest/icons/heroicons'
2. Use Qwik's `$()` function to create a serialized version
3. Export it from the icons file

Example:

```tsx
// In icons.ts
import { $ } from "@builder.io/qwik";
import { HiNewIconOutline } from "@qwikest/icons/heroicons";

export const NewIcon = $(HiNewIconOutline);
```

## Dark Mode Support

Icons automatically adapt to dark mode by inheriting colors from the text context.
