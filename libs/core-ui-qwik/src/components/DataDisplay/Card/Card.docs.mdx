# Card

The Card component provides a flexible container for displaying content with consistent styling. It supports various visual styles, elevations, and interactive states.

## Features

- Multiple elevation (shadow) levels
- Various border radius options
- Different style variants: default, outlined, filled
- Interactive hover effects
- Support for media content with optional overlays
- Structured layout with header, body, and footer components
- Accessibility features
- RTL support

## Usage

### Basic Card

```tsx
import { component$ } from "@builder.io/qwik";
import { Card, CardBody } from "@/components/Core/DataDisplay/Card";

export default component$(() => {
  return (
    <Card>
      <CardBody>
        <h3 class="mb-2 text-lg font-medium">Basic Card</h3>
        <p>This is a simple card with just a body and some content.</p>
      </CardBody>
    </Card>
  );
});
```

### Card with Header, Body, and Footer

```tsx
import { component$ } from "@builder.io/qwik";
import {
  Card,
  CardHeader,
  CardBody,
  CardFooter,
} from "@/components/Core/DataDisplay/Card";

export default component$(() => {
  return (
    <Card elevation="md" radius="lg">
      <CardHeader withBorder>
        <h3 class="text-lg font-medium">Card Title</h3>
      </CardHeader>
      <CardBody>
        <p>Card content goes here. This is the main content area.</p>
      </CardBody>
      <CardFooter withBorder alignRight>
        <button class="rounded-md bg-primary-500 px-4 py-2 text-white">
          Action
        </button>
      </CardFooter>
    </Card>
  );
});
```

### Card with Media

```tsx
import { component$ } from "@builder.io/qwik";
import {
  Card,
  CardMedia,
  CardBody,
  CardFooter,
} from "@/components/Core/DataDisplay/Card";

export default component$(() => {
  return (
    <Card interactive hoverEffect="raise">
      <CardMedia
        src="https://example.com/image.jpg"
        alt="Example image"
        height="200px"
      />
      <CardBody>
        <h3 class="mb-2 text-lg font-medium">Card with Media</h3>
        <p>This card includes an image at the top.</p>
      </CardBody>
      <CardFooter>
        <button class="text-primary-600">Learn More</button>
      </CardFooter>
    </Card>
  );
});
```

### Interactive Card as Link

```tsx
import { component$ } from "@builder.io/qwik";
import { Card, CardBody } from "@/components/Core/DataDisplay/Card";

export default component$(() => {
  return (
    <Card
      interactive
      hoverEffect="shadow"
      href="https://example.com"
      target="_blank"
    >
      <CardBody>
        <h3 class="mb-2 text-lg font-medium">Link Card</h3>
        <p>This entire card is a link. Click it to navigate.</p>
      </CardBody>
    </Card>
  );
});
```

### Card Subcomponents

#### CardHeader

The `CardHeader` component is used for the top section of a card.

| Prop              | Type        | Default | Description                           |
| ----------------- | ----------- | ------- | ------------------------------------- |
| children          | JSXChildren |         | Content to be displayed in the header |
| class             | string      |         | Additional CSS classes                |
| withBorder        | boolean     | false   | Whether to display a bottom border    |
| withBottomPadding | boolean     | true    | Whether to include bottom padding     |

#### CardBody

The `CardBody` component is used for the main content area of a card.

| Prop        | Type        | Default | Description                         |
| ----------- | ----------- | ------- | ----------------------------------- |
| children    | JSXChildren |         | Content to be displayed in the body |
| class       | string      |         | Additional CSS classes              |
| withPadding | boolean     | true    | Whether to include padding          |

#### CardFooter

The `CardFooter` component is used for the bottom section of a card, typically containing actions.

| Prop           | Type        | Default | Description                           |
| -------------- | ----------- | ------- | ------------------------------------- |
| children       | JSXChildren |         | Content to be displayed in the footer |
| class          | string      |         | Additional CSS classes                |
| withBorder     | boolean     | false   | Whether to display a top border       |
| withTopPadding | boolean     | true    | Whether to include top padding        |
| alignRight     | boolean     | false   | Whether to align content to the right |

#### CardMedia

The `CardMedia` component is used to display images or other media within a card.

| Prop           | Type                                                     | Default              | Description                                 |
| -------------- | -------------------------------------------------------- | -------------------- | ------------------------------------------- |
| src            | string                                                   |                      | Source URL of the media (required)          |
| alt            | string                                                   | ""                   | Alternative text for the media              |
| class          | string                                                   |                      | Additional CSS classes                      |
| height         | string                                                   | "200px"              | Height of the media                         |
| width          | string                                                   | "100%"               | Width of the media                          |
| objectFit      | "cover" \| "contain" \| "fill" \| "none" \| "scale-down" | "cover"              | Object fit property for the media           |
| isTop          | boolean                                                  | true                 | Whether the media is at the top of the card |
| overlay        | JSXChildren                                              |                      | Overlay content to display over the media   |
| overlayOpacity | number                                                   | 0.3                  | Opacity of the overlay background           |
| overlayColor   | string                                                   | "rgba(0, 0, 0, 0.5)" | Background color of the overlay             |

## Accessibility

The Card component is built with accessibility in mind:

- Proper semantic structure with appropriate ARIA attributes
- When used as a link or button, includes proper keyboard focus handling
- Clearly indicates interactive state with visual cues
- Ensures sufficient color contrast for text content
- Supports keyboard navigation

## Best Practices

- Use cards to group related content and actions
- Keep card content concise and focused on a single topic
- Use appropriate elevation to establish visual hierarchy
- For interactive cards, make sure the entire card is clickable, not just parts of it
- Include clear visual cues for interactive cards
- Use consistent padding and spacing within cards
- Consider using the `fullHeight` prop when displaying cards in a grid to maintain consistent heights
- Avoid nesting cards within cards
- Use CardMedia component for images and other media content
