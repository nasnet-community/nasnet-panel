# Breadcrumbs

The Breadcrumbs component provides hierarchical navigation, showing users their current location within the application.

## Overview

Breadcrumbs are a navigation pattern that:

- Show the user's current location in a hierarchical structure
- Provide a way to navigate back to higher-level pages
- Help users understand the site structure
- Enhance accessibility with proper ARIA attributes

The Breadcrumbs component is fully accessible, following the [W3C WAI-ARIA Breadcrumb Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/breadcrumb/).

## Usage

```tsx
import { Breadcrumbs } from '@connect/Core/Navigation/Breadcrumbs';
import type { BreadcrumbItem } from '@connect/Core/Navigation/Breadcrumbs';

// Define your breadcrumb items
const items: BreadcrumbItem[] = [
  {
    label: 'Home',
    href: '/',
  },
  {
    label: 'Products',
    href: '/products',
  },
  {
    label: 'Current Page',
    isCurrent: true, // Mark the current page
  },
];

// Basic usage
<Breadcrumbs items={items} />

// With custom separator
<Breadcrumbs
  items={items}
  separator=">"
/>

// Responsive behavior (collapses on small screens)
<Breadcrumbs
  items={items}
  maxItems={3}
  expanderLabel="..."
/>
```

## Accessibility

The Breadcrumbs component follows accessibility best practices:

- Uses semantic `<nav>` and `<ol>` elements
- Includes `aria-label` for the navigation region
- Applies `aria-current="page"` to the current page item
- Makes separators invisible to screen readers with `aria-hidden="true"`
- Ensures keyboard navigation for interactive links
